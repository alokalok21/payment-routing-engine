# Interchange Rate Model — Model Card

## Overview

A gradient-boosted regression model that predicts the interchange rate (%) charged
by a payment scheme for a given card transaction. Used as a tool by the Bedrock
routing agent to compare interchange cost across eligible schemes during routing.

## Inputs (Features)

| Feature           | Type            | Encoding         |
|---                |---              |---               |
| `scheme_id`       | categorical (7) | one-hot          |
| `card_type`       | categorical (3) | one-hot          |
| `card_product`    | categorical (4) | one-hot          |
| `mcc`             | categorical (10)| one-hot          |
| `merchant_country`| categorical (10)| one-hot          |
| `card_country`    | categorical (10)| one-hot          |
| `amount`          | numeric         | raw              |
| `cross_border`    | boolean → int   | 0/1              |

Total: 46 encoded features (44 one-hot + 2 numeric).

## Output

| Output             | Type    | Range       |
|---                 |---      |---          |
| `interchange_rate` | numeric | 0.05% – 4.5%|

## Architecture

| Property      | Value                          |
|---            |---                             |
| Algorithm     | XGBoost regression             |
| Trees         | 300                            |
| Max depth     | 6                              |
| Learning rate | 0.08                           |
| Subsample     | 0.9                            |
| Tree method   | `hist`                         |
| Loss          | squared error                  |

## Training Data

Synthetic data generated by `generate_synthetic_data.py` with a fixed seed (42).
25,000 rows. Each row simulates one historical settled transaction with a "true"
interchange rate computed from a parametric model:

```
rate = base[scheme, card_type]
     × product_multiplier[card_product]
     × mcc_multiplier[mcc]
     × (cross_border_multiplier if merchant != card_country else 1.0)
     × (1 + small_amount_premium)
     + gaussian_noise(0, 0.05)
```

The parametric base rates are loosely modelled on published interchange tier
structures (EU-regulated domestic, US unregulated, AMEX premium, UnionPay
domestic). This is *not* real settlement data — it is deliberately synthetic to
isolate the AI/ML routing contribution from access-to-real-data concerns.

## Evaluation

Held-out 20% test split. Metrics from the most recent training run:

| Metric | Value   |
|---     |---      |
| MAE    | 0.0393  |
| RMSE   | 0.0543  |
| R²     | 0.9988  |

These numbers are very high because the underlying data-generation process is
nearly deterministic (small noise). On real-world transaction data, R² of 0.7–
0.85 would be more realistic.

## Limitations

- Synthetic training data; not validated against real interchange invoices
- Country vocabulary fixed to 10 ISO-2 codes
- MCC vocabulary fixed to 10 MCCs
- `handle_unknown="ignore"` in the one-hot encoder means unseen values
  contribute zero to the feature vector — predictions for unknown
  schemes/countries/MCCs will be biased toward the training mean
- Amount is treated as a raw scalar; extreme values outside the lognormal
  training distribution may produce unreliable estimates
